Documentation API

Integrez Peppol Bridge a votre logiciel comptable en 30 minutes. Deux methodes : CSV import (le plus simple) ou Webhook JSON (temps reel).

Demarrage rapide en 3 etapes

  1. Creez un compte — POST /api/tenants/register pour obtenir votre API key
  2. Testez en CSV — POST /api/invoices/import-csv avec ?send=false
  3. Passez en production — retirez ?send=false ou utilisez le webhook

Authentification

Tous les endpoints API requierent une API key dans le header X-API-Key.

POST /api/tenants/register

Cree un nouveau tenant et retourne une API key.

Request Body (JSON)
{ "name": "Mon Editeur SAS", "email": "tech@monediteur.fr" }
Response 200
{ "api_key": "pb_a1b2c3d4e5f6...", "message": "Store this API key. It won't be shown again.", "name": "Mon Editeur SAS", "hint": "Use header X-API-Key: <api_key> on all API calls" }
⚠️ Stockez cette API key. Elle ne sera plus jamais affichee.
GET /api/tenants/me

Recupere les informations du tenant courant.

Header
X-API-Key: pb_a1b2c3d4e5f6...

Envoi direct JSON (existant)

L'endpoint d'origine pour envoyer une facture directement.

POST /invoices/send

Meme format JSON que le webhook. Ne necessite pas d'API key (utilise les credentials .env).

POST /invoices/build-xml

Construit l'XML UBL sans l'envoyer. Utile pour la validation.

POST /invoices/validate

Valide et sauvegarde la facture dans l'historique local.

Sandbox / Tests

Pour tester sans envoyer de factures reelles sur le reseau Peppol :

  • Utilisez POST /api/invoices/import-csv?send=false pour valider uniquement
  • Utilisez POST /invoices/build-xml pour generer l'XML sans envoi
  • Les factuves de test sont sauvegardees dans invoice_history.json
⚠️ Endpoint_id Peppol : le endpoint_id doit etre un identifiant Peppol valide (ex: 9925:FR12345678900012). En sandbox, utilisez des IDs de test fournis par votre SMP.

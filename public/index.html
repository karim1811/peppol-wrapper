Cas d'usage

Vous êtes éditeur d'un ERP, d'un SaaS de gestion ou d'un outil métier ? Vous voulez proposer la e-facturation Peppol à vos clients sans développer un connecteur from scratch ? Peppol Bridge est bâti pour ça.

Intégration simple : une API, pas un logiciel.

1. Récupérez la donnée métier

Nom, adresses, ID endpoint, lignes de facture : envoyez-la depuis votre existant.

2. Transformez en UBL conforme

Peppol Bridge convertit vos données en document conforme Peppol, prêt à être échangé.

3. Envoyez dans le réseau

Transmission au destinataire Peppol sans gestion de protocole propriétaire.

Exemple d'appelPOST /api/invoices/send
{ "seller": { "name": "Acme", "endpoint_id": "9925:acme" }, "buyer": { "name": "Client", "endpoint_id": "9925:client" }, "invoice": { "number": "INV-2026-001", "issue_date": "2026-06-11" }, "lines": [ { "description": "Prestation", "quantity": 1, "unit_price": 100, "tax_percent": 20 } ] }
200 OKEnvoi confirmé / erreur détaillée
